Что такое Go by Example и зачем он нам

Go by Example отлично закрывает синтаксис и базовые идиомы языка на маленьких живых примерах. Мы берём его как маршрут по темам, а я в любой момент рядом, если что-то будет непонятно.

✓
Даёт крепкую базу: переменные, функции, структуры, интерфейсы, горутины, каналы
✓
Позволяет быстро "пощупать" каждую тему на коде
!
Не заменяет прод-практику: архитектуру, тестирование, профилирование мы наращиваем отдельно
Идея: сначала понять тему на примере, потом закрепить своей задачей в этой же теме. Я рядом везде и всегда, и всегда готов помочь.

Шаг 1 — Создай общий репозиторий

Создай новый репозиторий на GitHub, например go-by-example-practice.

  1. Зайди на github.com/new
  2. Owner: твой аккаунт, Repository name: go-by-example-practice
  3. Выбери Public (пусть репозиторий будет открытым)
  4. В поле Description напиши всё что хочешь, тут только твоя фантазия
  5. Поставь галочку Add a README file
  6. Нажми Create repository

Шаг 2 — Пригласи меня как контрибьютера

Добавь меня в репозиторий, чтобы я мог пушить в него изменения и быть рядом с тобой в этом обучении каждый день.

  1. Открой репозиторий → Settings
  2. Слева выбери Collaborators and teams
  3. Нажми Add people
  4. Введи мой ник: gopherchan2006
  5. Выбери аккаунт и нажми Add / Invite

После этого я принимаю инвайт:

  1. Открываю уведомления на GitHub (иконка колокольчика)
  2. Нахожу приглашение в репозиторий
  3. Нажимаю Accept invitation
После принятия у меня появится право делать коммиты и пушить в этот репозиторий, и мы будем проходить всё вдвоем, спокойно и с заботой.

Шаг 3 — Структура папок по темам

Я заранее закоммичу структуру папок и задания под каждую тему с Go by Example. Мы будем проходить их по очереди, а решения хранить каждый в своей папке.

Формат выполненного задания я опишу внутри файла задач. И еще важный момент: мы периодически возвращаемся к старым темам, поэтому вместо одного tasks.md будут файлы по датам, например tasks30_04_2026.md, tasks15_05_2026.md и так далее. В этих же файлах я буду писать дедлайны — до какой даты нужно закончить каждую задачу. В своей подпапке называй файлы так же, как называется задача, только с расширением .go. Если для одной задачи нужно несколько файлов, создай подпапку с названием задачи, например tasks_30_04_2026.

Шаг 4 — Как работать над темой

1
Открываешь тему на gobyexample.com и не просто читаешь: лапками переписываешь код вручную, запускаешь, смотришь вывод и анализируешь, какая команда за что отвечает
2
Я добавляю задания в файл по дате, например topics/xx-topic/tasks30_04_2026.md
3
Ты решаешь задания в vikasuperpuper/, я решаю в gopherchan2006/
4
Каждый делает коммит со своим решением
5
Потом разбираем вместе: что получилось, где было сложно, как улучшить. Нейронку спрашивай только когда совсем полный затуп и нужно разобрать тему; в остальное время лучше дополнительно идти в интернет и углублять тему — так прогресс будет сильнее. GitHub Copilot у тебя под рукой как ассистент
Главное правило: не гонимся за скоростью. Лучше меньше тем, но с полным пониманием и практикой. Я рядом всегда, правда в любой момент.
